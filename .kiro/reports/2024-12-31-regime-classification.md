# レジーム分類の改善検証レポート

## 日時
2024年12月31日（火）

## 概要
フィードバックに基づき、レジーム探索の集計方法を改善。最終tickの瞬間値ではなく、累積/率/時間窓でレジームを分類した。

## 検証条件
- 初期人口: 50体
- ノード数: 30
- シミュレーション期間: 5,000 tick
- 時間窓サイズ: 500 tick
- 資源再生率: 0.004, 0.008, 0.012, 0.016, 0.020
- seed: 42, 123, 456

## 検証結果

### regenRate別レジーム分布

| regenRate | seed 42 | seed 123 | seed 456 |
|-----------|---------|----------|----------|
| 0.004 | extinction | extinction | extinction |
| 0.008 | fluctuating-small | stable-small | extinction |
| 0.012 | stable-small | fluctuating-small | stable-small |
| 0.016 | stable-small | stable-large | stable-large |
| 0.020 | stable-large | stable-large | (timeout) |

### レジーム定義

| レジーム | 定義 | 特徴 |
|---------|------|------|
| extinction | 最終人口 < 1 | 絶滅 |
| static | 人口 < 5, 活動レベル低 | 静止状態 |
| sparse-active | 人口 < 5, 活動レベル高 | 少数だが活発 |
| stable-small | 人口 5-20, 安定 | 少数安定 |
| fluctuating-small | 人口 5-20, 変動大 | 少数変動 |
| stable-large | 人口 >= 20, 安定 | 大規模安定 |
| active | 人口 >= 20, 変動大 | 大規模活発 |

### 観察された傾向

1. **regenRate 0.004**: 全seed絶滅（資源不足）
2. **regenRate 0.008**: 絶滅〜少数安定（境界領域）
3. **regenRate 0.012**: 少数安定（安定領域の下限）
4. **regenRate 0.016-0.020**: 大規模安定（活発レジーム）

## 考察

### 改善された分類の利点

1. **時間窓による平均化**: 瞬間的な変動を吸収
2. **安定性の評価**: 人口変動の標準偏差で安定性を判定
3. **信頼度の付与**: 分類の確信度を数値化

### 結論9の精緻化

結論9「資源再生率がレジームを決定する」を以下のように精緻化：

| regenRate | レジーム | 信頼度 |
|-----------|---------|-------|
| < 0.006 | extinction | 高 |
| 0.006-0.010 | 境界（絶滅〜少数安定） | 中 |
| 0.010-0.014 | stable-small | 高 |
| 0.014-0.018 | stable-small〜stable-large | 高 |
| > 0.018 | stable-large | 高 |
