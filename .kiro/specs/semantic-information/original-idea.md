# 情報の意味の導入（Semantic Information）

**日時**: 2024年12月31日

## 背景

情報伝達機能を実装し、エンティティのstate充填率は50%→93.2%に増加した。しかし、知識ボーナスの適用率は0.4%と低く、情報が行動に与える影響は限定的である。

これは、stateとdataの類似度が偶然に依存しているためである。情報に「意味」を持たせることで、知識ボーナスの効果を高め、Q2（技術の蓄積）をより完全に達成する。

## アイデア

stateの特定パターンが行動効率に影響を与える仕組みを導入する。

### 案1: スキルパターン

stateの一部を「スキル」として解釈し、特定の行動の効率を上げる。

- stateの最初の8バイトを「スキルベクトル」として解釈
- 各バイトが特定の行動（Harvest, Repair, Create等）の効率に対応
- 例: スキルベクトル[0]が高い → Harvest効率が上がる

### 案2: 環境適応パターン

stateの一部を「環境適応」として解釈し、特定の環境での生存効率を上げる。

- stateの一部がノードの属性（温度、地形）との適合度を決定
- 適合度が高いほど維持コストが下がる

### 案3: アーティファクト専門化

stateの一部を「専門知識」として解釈し、特定のアーティファクトの修復効率を上げる。

- アーティファクトのdataとstateの類似度が高いほど修復効率が上がる（既存の知識ボーナス）
- 追加: stateの特定パターンが「専門分野」を示し、同じ専門分野のアーティファクトを優先的に修復

## 選択: 案1（スキルパターン）

最もシンプルで、既存の行動システムと自然に統合できる。

## 非恣意性チェック

- [ ] 新しい公理を追加する必要があるか？ → NO（既存の公理8, 9の接続強化）
- [ ] 特定の成果を誘導していないか？ → NO（スキルの効果は物理的に動機付けられる）
- [ ] 観測のみの原則に沿っているか？ → YES（スキル効果はログとして記録）

## 期待される効果

1. 知識ボーナス適用率の向上（0.4% → 目標10%以上）
2. 情報の「意味」が生存に影響を与えることで、情報の自然選択が起きる
3. 特定のスキルパターンが集団内で広まる可能性（文化の萌芽）
